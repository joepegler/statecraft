# Wallets & Balance Fixtures

**Test runners:** Examples use Vitest. `scenario(...)` works with any runner that accepts an async test callback (Jest, Node `node:test`, and others).

Wallet and balance fixtures prepare the scenario account that later steps (token seeding and contract interactions) will use.

Choose between:
- `withFundedWallet` when you need a fresh deterministic actor with known balances
- `withImpersonation` when your test must behave as an existing on-chain actor (admin, treasury, whale) without private key access

## Fixtures

- [`withFundedWallet({ balance[, privateKey, erc20] })`](/fixtures/wallets/withFundedWallet)
- [`withImpersonation({ address[, balance, stopOnExit] })`](/fixtures/wallets/withImpersonation)
