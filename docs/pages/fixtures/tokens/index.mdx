# Tokens & ERC-20 Balance Fixtures

**Test runners:** Examples use Vitest. `scenario(...)` works with any runner that accepts an async test callback (Jest, Node `node:test`, and others).

Use these fixtures to seed ERC-20 balances in compatible local and forked runtimes (for example, Anvil-style nodes).

This is most useful when token mint mechanics are incidental to the test and you want to start directly from business preconditions like "user has spendable token balance."
