# Statecraft

Composable scenario runtime for Ethereum integration tests in Vitest, built on viem and local Anvil runtimes.

## Who it is for

Statecraft fits TypeScript teams that want deterministic EVM tests: pinned forks (`rpcUrl` + `blockNumber`), reproducible balances, and explicit setup—without copy-pasting chain bootstrapping into every test file.

## Problems it avoids

- One-off local chain / fork wiring in each test.
- Ad hoc wallet funding and RPC client creation.
- Brittle test suites that depend on implicit global setup instead of named fixtures.

## What you get instead

You compose scenario steps (`withChain`, `withFork`, `withFundedWallet`, `withContracts`, `withDeployments`, …) that attach typed context; the test body stays focused on behavior and assertions. See [Core concepts](/core-concepts) for how steps chain together.

## Package layout

- `packages/runtime`: runtime lifecycle (start/stop local chain or fork)
- `packages/clients`: viem `publicClient`, `walletClient`, `testClient` creation
- `packages/scenarios`: scenario engine and `withX` fixtures
- `packages/vitest`: ergonomic Vitest-facing exports
- `packages/examples`: runnable demonstrations
